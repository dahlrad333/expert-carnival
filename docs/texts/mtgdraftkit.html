Poker Timer

  • Summary

    Currently working on a Android app to replace the one my play group uses as it has some deficiencies.

  • Methods

    Android, HTML5/CSS3, wireframing

  • Client

    Personal project (notwithstanding all of my poker buddies that get to test it as I make updates)
    2012

THE PROBLEM

From time to time, I enjoy playing poker. One of the formats in which poker can be played is tournament style. In this style, there is a blind schedule which is to be followed. This requires some sort of timing device. While in theory a regular clock would be sufficient, we desire a more full featured tool specifically designed.

THE GOAL

To be more functional than any of the current free apps available in the Play store. This will allow my play group to have a better experience while we are playing.

THE PROCESS

To accomplish this, I am using a framework to help build the UI for Android quickly. A very very very very very rough proof-of-concept, is currently on my GitHub if you are clever and know how to find it. To be honest, I really have not spent much time on it. I usually say I am going to have it ready next week, and then I get busy with other stuff and it does not happen. Maybe this week.
